# Contributing to Hort

Small, focused contributions are welcome.

## Before opening a pull request

1. Create a branch from `main`.
2. Keep captured data and local database files out of commits, tests, issues,
   screenshots, and logs.
3. Run `Scripts/check.sh`.
4. Update tests and documentation when behavior changes.
5. Describe the user impact and privacy impact in the pull request.

Use fabricated test data only. Never add real clipboard contents, screenshots,
tokens, usernames, home-directory paths, or generated app data to the repository.

Bug reports and feature requests should use the repository's issue forms.
Security problems must be reported through a private security advisory as
described in [SECURITY.md](SECURITY.md).
